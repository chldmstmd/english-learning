# 产品需求文档 (PRD)

**项目名称：** Context-Aware Smart Reader (暂名)
**版本：** V1.0 (核心：单词交互学习系统)
**目标：** 通过点击式交互，实现基于语境的单词翻译、跨文章生词同步及记忆管理。

---

## 1. 业务逻辑与用户故事

### 1.1 核心流程描述

用户上传或粘贴文章进行阅读。点击单词后，系统不仅仅查词，还会利用 LLM (大语言模型) 判断该词在当前句子中的确切含义，并触发全量文章同步。

### 1.2 状态机定义 (Word Status)

单词在系统中存在三种状态，通过点击循环切换：

- **状态 0 (无标注/Default)：** 黑色。用户未标记或已掌握。
- **状态 1 (生词/New)：** 红色。单词上方显示 Context 翻译，进入生词表。
- **状态 2 (巩固中/Reviewing)：** 黄色。标注消失，仅保留背景色，提醒用户回想含义。

---

## 2. 功能详细说明 (V1.0)

### F1.1 语境感知翻译 (Contextual Annotation)

- **触发：** 用户单击正文中的单词。
- **动作：** 发送请求至后端，后端结合当前句子内容调用 AI。
- **展现：** 在该单词正上方（类似拼音注音/Ruby 格式）显示简短的中文翻译。
- **全局同步：** 该单词在用户库中所有文章的实例立即变为红色并带上翻译。

### F1.2 动态侧边栏 (Detailed Analysis)

展示内容：点击单词时，右侧面板弹出详细信息，包括：

- 音标及发音按钮。
- AI 生成的"当前语境解析"。
- 常规释义及例句。

### F1.3 生词表管理 (Vocab Database)

- **自动入库：** 变为"红色"的词自动存入个人的 Vocabulary_Table。
- **字段预留：** 单词、词性、语境翻译、原始句子、状态代码 (1/2/0)、创建时间。

### F1.4 交互式状态切换

- **Red → Yellow：** 再次单击红色单词。翻译标注移除，颜色转黄。生词表状态更新为"巩固中"。
- **Yellow → Default：** 单击黄色单词。背景色消失，恢复普通文本。生词表逻辑删除或标记为"已掌握"。

## 3. 架构设计考虑 (面向未来 F2.1)

为了支持未来的 "句子结构可视化"，V1.0 的架构需要做以下前瞻设计：

### 3.1 文本处理层 (NLP Engine)

- **预处理：** 文章存入数据库前，不能只存长字符串。必须经过 Tokenization (分词) 流程。
- **数据结构：** 建议将文章存储为有序的 JSON 数组。
  - 每个词是一个对象 `{text: "Blacks", pos: "NNP", index: 12}`。
  - **理由：** 未来句子分析需要定位句子的 Dependency Tree (依存关系)，只有索引化了才能准确在右下角画出结构图。

### 3.2 接口预留 (API Design)

- 预留 `/api/v1/analyze-sentence` 接口。
- 前端拉选句子时，获取起始词索引 `start_index` 和结束词索引 `end_index`。

### 3.3 前端画布预留 (UI/UX)

- **右下角容器：** 采用响应式布局。V1.0 期间该区域可隐藏或显示占位引导。
- **渲染选型：** 建议前端引入 Canvas 或 SVG 库（如 D3.js 或 Cytoscape.js），这些工具既能处理简单的生词分析，也能胜任未来的复杂语法树绘制。

---

## 4. 技术栈推荐 (建议)

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 前端 | React.js + Tailwind CSS | 易于处理复杂的单词状态样式 |
| 后端 | Python (FastAPI/Flask) | 处理句子结构图需要强大的 Python NLP 库（SpaCy / Stanza） |
| AI | OpenAI GPT-4o 或 Gemini 1.5 Pro | 擅长根据 context 返回特定翻译，并能输出句子结构的 JSON 格式 |
| 数据库 | PostgreSQL | 利用其强大的 JSONB 格式存储 Token 化的文章 |

---

## 5. 验收标准 (Acceptance Criteria)

- [ ] 单击文章中的 "bank"，若语境是河流，标注为"河岸"而非"银行"。
- [ ] 在文章 A 标记 "access" 为红色，打开文章 B，所有 "access" 必须自动呈现红色且带翻译。
- [ ] 点击红色词后，右侧面板延迟不得超过 1s。
- [ ] 状态切换逻辑 (红 -> 黄 -> 无) 准确无误。

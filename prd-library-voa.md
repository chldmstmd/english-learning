# 产品需求文档 (PRD)

**功能名称：** 内容库 — VOA Learning English 集成  
**版本：** V1.1  
**依赖：** 基于 V1.0 核心单词交互系统之上扩展  
**目标：** 为用户提供开箱即用的英文阅读材料库，无需自行寻找或粘贴文章，同时保留 V1.0 的所有生词交互能力。

---

## 1. 背景与动机

V1.0 支持用户自行上传/粘贴文章。用户的核心痛点之一是：**找到合适难度的英文阅读材料本身就是障碍**，尤其对初中级学习者。

VOA Learning English（美国之音英语学习频道）由美国联邦政府出资，内容属公共领域，无版权顾虑。其文章具有以下优势：
- 难度分级明确（Level 1 慢速英语 / Level 2 标准英语）
- 题材多样（新闻、科技、健康、文化等）
- 内容持续更新（每日发布）
- 提供 RSS Feed，可程序化获取

| # | 用户故事 |

## 3. 功能范围

### 3.1 包含 (In Scope, V1.1)

- 后台定时同步 VOA RSS，将新文章写入系统内容库
- 前端"内容库"入口页，支持按话题/难度/时间排序浏览
- 用户从内容库打开文章后，使用与 V1.0 完全相同的阅读与生词交互界面
- 内容库文章与用户自建文章在生词库中共享同一份单词状态（一处标红，处处同步）

### 3.2 不包含 (Out of Scope, V1.1)

- VOA 音频播放（文章配套 MP3）
- 基于生词表的个性化推荐（属 F2.2，Post-MVP）
- 用户收藏/书签功能
- 离线缓存
- 其他内容源（Project Gutenberg、Wikipedia 等）

## 4. 数据模型扩展

### 4.1 新增字段：文章来源标识

在现有 `Article` 表新增以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `source` | enum | `user_upload` / `voa` （默认 `user_upload`，兼容 V1.0） |
| `source_url` | string \| null | 原始文章 URL（VOA 文章填入，用户上传为 null） |
| `source_category` | string \| null | VOA 话题分类，如 `science-technology`、`health-lifestyle` |
| `difficulty` | enum \| null | `level1`（慢速）/ `level2`（标准）/ null（用户上传不标注） |
| `published_at` | timestamp \| null | VOA 原文发布时间（来自 RSS `<pubDate>`） |
| `cover_image_url` | string \| null | 文章封面图（来自 RSS `<media:content>`，可选） |
| `is_library` | boolean | `true` = 内容库公共文章；`false` = 用户私有文章 |

**兼容性说明：** 所有新增字段对 V1.0 用户上传文章均为 null 或默认值，不破坏现有数据。

### 4.2 新增表：同步状态记录

新增 `VoaSyncLog` 表，用于追踪每次 RSS 同步：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | integer | 主键 |
| `feed_url` | string | 同步的 RSS 地址 |
| `synced_at` | timestamp | 本次同步时间 |
| `new_articles` | integer | 本次新增文章数 |
| `status` | enum | `success` / `failed` |
| `error_message` | string \| null | 失败时的错误信息 |

## 5. VOA RSS 数据源

### 5.1 目标 Feed 列表（初始范围）

VOA Learning English 提供按栏目分类的 RSS。以下为 V1.1 初始同步的 Feed：

| 栏目 | 难度 | RSS URL |
|------|------|---------|
| Science & Technology | Level 2 | `https://learningenglish.voanews.com/api/z_qyql$ypk` （需核实具体 RSS 路径）|
| Health & Lifestyle | Level 2 | 同上，按栏目 |
| US History | Level 1 | 慢速英语频道 |
| Words & Stories | Level 1 | 词汇学习专栏 |

> **实施注意：** VOA 官方 RSS 地址需在开发前从 learningenglish.voanews.com 各栏目页实际提取（页面 `<link rel="alternate">` 标签），以上为示意。初始阶段建议选取 3-5 个话题栏目，避免冷启动时内容库过于空洞或重复。

### 5.2 RSS 字段映射

| RSS 字段 | 映射到 Article 字段 | 备注 |
|---------|-------------------|------|
| `<title>` | `title` | 文章标题 |
| `<description>` / `<content:encoded>` | 正文提取来源 | 需清洗 HTML 标签，提取纯文本 |
| `<pubDate>` | `published_at` | 解析为 UTC timestamp |
| `<link>` | `source_url` | 原文链接，用于去重判断 |
| `<media:content url=...>` | `cover_image_url` | 封面图（可选） |
| Feed 本身的栏目标识 | `source_category` | 由同步任务写入 |

### 5.3 内容清洗规则

RSS 正文通常含 HTML 标签、广告推广链接及版权声明尾注。清洗步骤：

1. 使用 HTML 解析器去除所有标签，保留纯文本
2. 删除文末固定格式的版权声明行（如 "I'm ___. Words in This Story: ..." 固定结尾区块）
3. 正文字数不足 100 词的条目跳过（通常为纯图片或活动预告）
4. 清洗后的正文进入 V1.0 的 Tokenization 流程（spaCy），结果存为 `content_tokens`

## 6. 同步机制

### 6.1 定时同步策略

| 维度 | 设计 |
|------|------|
| 触发方式 | 后台定时任务（cron），每 **6 小时**执行一次 |
| 去重逻辑 | 以 `source_url` 为唯一键，已存在则跳过 |
| 追量限制 | 单次同步每个 Feed 最多拉取 **20 条**最新文章，避免首次初始化数据量过大 |
| 失败处理 | 单篇文章 Tokenization 失败时记录错误并跳过，不影响本批次其他文章；整体同步结果写入 `VoaSyncLog` |
| 手动触发 | 管理接口 `POST /admin/sync-voa` 支持手动立即触发同步（开发/运维用途） |

### 6.2 内容库容量规划（V1.1）

- 初始：从历史 RSS 中回填近 30 天文章（每个 Feed 最多 50 篇）
- 稳定运行后：每日新增约 5-10 篇，按 `published_at` 排序展示最新内容
- V1.1 不设删除旧内容机制；若存储压力增大，在后续版本引入过期清理策略

## 7. 前端界面设计

### 7.1 导航入口

在主导航新增 **"Library（内容库）"** 标签，与现有 **"My Articles（我的文章）"** 并列。

### 7.2 内容库列表页

**筛选区（顶部）：**
- 话题：全部 / Science & Technology / Health / US History / Words & Stories / ...
- 难度：全部 / Level 1（慢速）/ Level 2（标准）
- 排序：最新发布 / 最多阅读（V1.1 仅实现"最新发布"）

**文章卡片（列表/网格）：**
- 标题
- 难度标签（Level 1 / Level 2）
- 话题分类标签
- 发布日期
- 预估阅读时长（基于字数估算，200 wpm）
- 封面图（若有）

**加载策略：** 分页加载，每页 20 篇，下滑加载更多。

### 7.3 阅读页（复用 V1.0）

用户点击内容库文章后，进入与 V1.0 完全相同的阅读界面，所有生词交互行为保持一致。

**差异点（相对于用户上传文章）：**
- 顶部增加"来源：VOA Learning English"标注及原文链接
- 不显示"删除文章"按钮（内容库文章为公共资源，不支持用户删除）
- 不显示"编辑标题"功能

## 8. 接口扩展

在现有接口基础上新增/调整：

| 接口 | 方法 | 说明 |
|------|------|------|
| `GET /api/v1/library` | GET | 获取内容库文章列表，支持 `category`、`difficulty`、`page`、`page_size` 参数 |
| `GET /api/v1/library/{article_id}` | GET | 获取单篇内容库文章（含 tokens） |
| `POST /admin/sync-voa` | POST | 手动触发 VOA 同步（需鉴权或仅限本地调用） |
| `GET /admin/sync-voa/logs` | GET | 查看同步历史记录 |

现有 `GET /api/v1/articles` 接口默认**仅返回用户自建文章**（`is_library=false`），不混入内容库文章，避免破坏现有我的文章列表。

## 9. 非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 内容库列表页首屏加载 ≤ 1s（数据库查询 + 分页，无实时外部请求） |
| 性能 | 定时同步任务（含 Tokenization）单次执行时间 ≤ 5 分钟 |
| 合规 | 文章展示须注明来源 "VOA Learning English" 并提供原文链接，符合公共域内容的署名惯例 |
| 容错 | VOA RSS 不可达时，同步任务静默失败并写 log，不影响应用其他功能 |
| 数据隔离 | 内容库文章不属于任何用户；生词状态仍为用户私有（与 V1.0 一致） |

## 10. 核心设计决策与权衡

### 决策 1：全量存储 vs 实时代理

**选择：** 将 VOA 文章**完整同步至本地数据库**，而非每次阅读时实时代理请求。

**理由：** Tokenization（spaCy）是 CPU 密集操作，不适合实时执行；本地存储保证了离线可用性，也方便生词状态的跨文章同步。

**代价：** 需维护同步任务，存储占用随文章数增长。

---

### 决策 2：共享生词库 vs 独立生词库

**选择：** 内容库文章与用户上传文章**共享同一份生词库**（`Vocabulary_Table`），单词状态全局统一。

**理由：** 这是 V1.0 的核心价值主张——"一处标红，处处同步"。内容库若孤立，会割裂学习连贯性。

---

### 决策 3：`is_library` 标志 vs 独立表

**选择：** 在现有 `Article` 表新增 `is_library` 标志，而非单独建 `LibraryArticle` 表。

**理由：** 阅读逻辑、Tokenization 结构、生词同步逻辑完全相同，分表会造成大量代码重复。单表通过标志区分，查询逻辑简单。

---

## 11. 验收标准

- [ ] 定时任务（或手动触发）能成功拉取 VOA RSS 并将新文章写入数据库，重复文章不重复写入。
- [ ] 内容库列表页按话题/难度筛选后，结果与数据库中 `source_category`、`difficulty` 字段一致。
- [ ] 点击内容库文章后，进入阅读页，生词交互行为与 V1.0 用户上传文章完全一致（点击标红、状态切换、侧边栏等）。
- [ ] 在内容库文章 A 中标记单词 "policy" 为红色后，用户上传文章 B 中的 "policy" 也自动呈现红色（生词库共享）。
- [ ] 内容库文章阅读页不显示"删除"按钮。
- [ ] 阅读页顶部显示 "来源：VOA Learning English" 及可点击的原文链接。
- [ ] VOA RSS 拉取失败时，`VoaSyncLog` 中记录失败状态，应用其他功能不受影响。
- [ ] `GET /api/v1/articles`（我的文章列表）不包含内容库文章。

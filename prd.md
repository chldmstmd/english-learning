# 产品需求文档 (PRD)

**项目名称：** Context-Aware Smart Reader (暂名)
**版本：** V1.0 (核心：单词交互学习系统)
**目标：** 通过点击式交互，实现基于语境的单词翻译、跨文章生词同步及记忆管理。

---

## 1. 业务逻辑与用户故事

### 1.1 目标用户

泛指需要阅读英文原文材料的中文用户，例如：备考 CET-6/GRE 的学生、阅读英文论文的研究人员、日常阅读英文资讯的职场人士。共同特征：有一定英文基础，但仍会遇到生词或多义词的语境理解困难。

### 1.2 用户故事

| # | 用户故事 |
|---|----------|
| US-01 | 作为一名英文阅读者，我希望点击不认识的单词时看到**当前句子语境下的翻译**，而不是词典里所有释义，以便快速理解句子意思而不打断阅读节奏。 |
| US-02 | 作为一名持续学习者，我希望在文章 A 标记的生词能在文章 B 中自动高亮，以便在不同语境中反复接触同一个词，加深记忆。 |
| US-03 | 作为一名复习中的学习者，我希望将生词切换为"巩固中"状态，让系统只提示颜色而不直接给出翻译，以便主动回想单词含义，练习记忆提取。 |
| US-04 | 作为一名学习者，我希望查看一个单词的音标、例句及 AI 语境解析，以便更全面地理解这个词的用法。 |

### 1.3 核心流程描述

用户上传或粘贴文章进行阅读。点击单词后，系统利用 LLM 判断该词在当前句子中的确切含义，在词上方展示语境翻译，并将该词同步标记至用户库中所有文章。

### 1.4 状态机定义 (Word Status)

单词存在四种逻辑状态。在文章中，**Unseen** 与 **Mastered** 视觉相同（均为黑色无标注），但在生词库中有本质区别。

```
[Unseen] ──点击──▶ [New/Red] ──点击──▶ [Reviewing/Yellow] ──点击──▶ [Mastered]
                       ▲                                                    │
                       └────────────────────点击────────────────────────────┘
```

| 状态 | 名称 | 文章中视觉 | 生词库记录 | 含义 |
|------|------|------------|------------|------|
| unseen | 未标记 | 黑色，无标注 | 无记录 | 从未被点击过 |
| new | 生词 | 红色 + 词上方语境翻译 | 有记录，status = `new` | 已识别为生词，需要学习 |
| reviewing | 巩固中 | 黄色背景，无翻译标注 | 有记录，status = `reviewing` | 用户需主动回想含义 |
| mastered | 已习得 | 黑色 + 浅灰色虚线下划线 | 有记录，status = `mastered` | 曾经学过，已掌握 |

**边界行为：**
- **文章加载时：** 生词库中状态为 `new` 或 `reviewing` 的词，在文章渲染后自动呈现对应颜色和标注；`mastered` 的词呈现浅灰色虚线下划线，不显示翻译注音。
- **Mastered → New 重入：** 点击文章中已习得（黑色）的词，该词重新进入 `new` 状态（变红 + 翻译），生词库记录更新，不新建词条。
- **"已习得"与"从未标记"的区分：** 文章中视觉相同，但生词库中 `mastered` 有历史记录（含习得时间），`unseen` 无记录，两者可用于统计学习进度。
- **文章删除时：** 仅删除该文章的语境翻译记录；词条本身及其在其他文章中的状态不受影响。

## 2. 功能详细说明 (V1.0)

### F1.1 语境感知翻译 (Contextual Annotation)

- **触发：** 用户单击正文中状态为 `unseen` 或 `mastered` 的单词。
- **动作：** 后端提取该单词所在句子，调用 AI 生成当前语境下的简短中文翻译。
- **展现：** 在该单词正上方以 Ruby 注音格式显示翻译（≤ 6 个中文字）。
- **全局同步策略：** 该词在用户库中所有文章的实例立即标为红色（状态 1）。各文章的翻译按各自语境独立生成：**已访问过的文章**在后台异步触发翻译生成；**未访问的文章**在用户首次打开时触发。不跨文章复用翻译，以保证语境准确性。
- **降级处理：** AI 接口超时（> 3s）时，展示通用词典释义作为兜底，并标注"(通用释义)"。

### F1.2 动态侧边栏 (Detailed Analysis)

- **触发：** 点击任意单词时，右侧面板滑入展示。
- **展示内容：**
  - 音标及发音按钮（TTS）。
  - AI 生成的当前语境解析（1-3 句话）。
  - 常规释义及例句（来自词典 API）。
- **reviewing 状态专属：** 单词处于 `reviewing` 状态时，侧边栏顶部始终展示完整语境翻译（作为"揭晓"入口），并在底部显示两个操作按钮：
  - **"已掌握"** — 将单词推进至 `mastered` 状态。
  - **"再复习一次"** — 状态保持 `reviewing` 不变，侧边栏关闭。
  - 若用户直接关闭侧边栏（未点击任何按钮），状态保持 `reviewing`。
- **关闭行为：** 点击正文空白区域或按 `Esc` 键关闭面板。
- **移动端适配：** 以底部抽屉（Bottom Sheet）形式展示，支持上滑展开/下滑收起。

### F1.3 生词表管理 (Vocab Database)

- **自动入库：** 单词首次变为红色（状态 1）时自动写入 `Vocabulary_Table`。
- **字段定义：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `word` | string | 单词原形（小写） |
| `pos` | string | 词性（名词/动词等） |
| `context_translation` | string | 入库时的语境翻译（词级存储，不区分义项；用户可通过重新标红更新） |
| `source_sentence` | string | 触发标注的原始句子 |
| `status` | enum | `new` / `reviewing` / `mastered` |
| `created_at` | timestamp | 首次入库时间（首次点击时） |
| `mastered_at` | timestamp | 最近一次进入 mastered 状态的时间 |
| `updated_at` | timestamp | 最近状态变更时间 |

### F1.4 交互式状态切换

| 操作 | 状态变化 | 视觉变化 | 数据库更新 |
|------|----------|----------|------------|
| 点击 unseen 词 | unseen → new | 变红 + 翻译标注出现 | 新建词条，status = `new` |
| 点击红色词 | new → reviewing | 翻译标注消失，背景变黄 | status = `reviewing` |
| 点击黄色词 | reviewing → reviewing（侧边栏打开） | 状态不变，侧边栏展示翻译 + 操作按钮 | 无变更 |
| 侧边栏点击"已掌握" | reviewing → mastered | 黄色消失，出现灰色虚线下划线 | status = `mastered`，记录 `mastered_at` 时间 |
| 侧边栏点击"再复习一次"或关闭 | reviewing → reviewing | 无变化 | 无变更 |
| 点击已习得词（灰色虚线） | mastered → new | 虚线消失，变红 + 翻译标注出现 | status = `new`，更新 `updated_at`，不新建词条 |

## 3. 接口约定

### 3.1 文章数据结构

文章存储前须经过 Tokenization 处理，以 JSON 数组形式存储，支持未来的句法分析扩展：

```json
[
  { "text": "The", "pos": "DT", "index": 0 },
  { "text": "bank", "pos": "NN", "index": 1 },
  { "text": "of", "pos": "IN", "index": 2 },
  { "text": "the", "pos": "DT", "index": 3 },
  { "text": "river", "pos": "NN", "index": 4 }
]
```

### 3.2 核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `POST /api/v1/translate-word` | POST | 传入单词 + 所在句子，返回语境翻译 |
| `GET /api/v1/vocab` | GET | 获取用户生词表 |
| `PATCH /api/v1/vocab/{word}/status` | PATCH | 更新单词状态 |
| `POST /api/v1/analyze-sentence` | POST | 预留：传入 `start_index` + `end_index`，返回句法分析结果 |

## 4. 非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 点击词后侧边栏内容加载延迟 ≤ 1s（正常网络环境下） |
| 性能 | 文章渲染时已标注词的颜色恢复 ≤ 500ms |
| 可用性 | AI 接口不可用时，系统仍可正常使用（降级至词典释义），不影响状态切换 |
| 数据隐私 | 用户上传的文章内容仅用于翻译请求，不用于模型训练；明确在用户协议中说明 |
| 规模边界 | 单篇文章最大 10,000 词；单用户生词表上限 10,000 条（V1.0） |
| 可访问性 | 红色/黄色状态须同时提供非颜色视觉区分（如下划线/图标），以支持色觉障碍用户 |

## 5. 技术栈推荐

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| 前端 | React.js + Tailwind CSS | 易于处理复杂的单词状态样式 |
| 后端 | Python (FastAPI) | NLP 生态成熟，便于未来扩展句法分析能力 |
| AI | OpenAI GPT-4o 或 Gemini 1.5 Pro | 擅长根据 context 返回特定翻译，可输出结构化 JSON |
| 数据库 | PostgreSQL | JSONB 格式适合存储 Token 化文章 |

> 具体库选型（NLP 库、图形渲染库等）在技术设计文档（TDD）中另行定义。

## 6. 未来规划 (Post-MVP Roadmap)

以下功能不在 V1.0 开发范围内，但架构设计需为其预留扩展空间。

### F2.1 间隔重复系统 (Spaced Repetition)

在生词表中引入 `interval`、`ease_factor`、`next_review_at` 字段，采用 SM-2 或 FSRS 算法驱动 `reviewing` 状态的触发时机，使复习调度与遗忘曲线对齐，而非依赖用户主动阅读触发。

**V1.0 预留：** `Vocabulary_Table` 字段设计须兼容上述字段的后续追加，避免破坏性迁移。

### F2.2 个性化文章推荐 (Vocabulary-Based Article Recommendation)

根据用户生词表（尤其是 `reviewing` 状态词）自动推荐包含这些词的文章，使用户在真实语境中反复接触目标词汇，强化间隔重复效果。

**V1.0 预留：** 文章的 Tokenization 结构（§3.1）已支持词级索引，可直接用于推荐系统的词-文章倒排索引构建。

### F2.3 句子结构可视化 (Sentence Dependency Visualization)

预留 `/api/v1/analyze-sentence` 接口（见 §3.2），支持用户选取句子后在右下角画布区渲染依存关系树。

## 7. 验收标准 (Acceptance Criteria)

- [ ] 单击文章中的 "bank"，若语境为河流，标注为"河岸"而非"银行"。
- [ ] 在文章 A 标记 "access" 为红色后，打开文章 B，所有 "access" 自动呈现红色；翻译按文章 B 的语境独立生成，而非沿用文章 A 的翻译。
- [ ] 点击红色词后，右侧面板内容在正常网络环境下延迟 ≤ 1s 展示。
- [ ] 点击黄色（reviewing）词后，侧边栏打开并展示完整翻译，单词状态保持黄色不变。
- [ ] 侧边栏点击"已掌握"后，单词变为黑色 + 灰色虚线下划线，`mastered_at` 写入数据库。
- [ ] 侧边栏点击"再复习一次"或直接关闭，单词保持黄色，数据库无变更。
- [ ] 点击文章中已习得（灰色虚线）的词，该词重新变红并触发语境翻译，生词库中不新建词条，`updated_at` 更新。
- [ ] mastered 与 unseen 词在文章中可通过灰色虚线下划线加以区分。
- [ ] AI 接口超时时，侧边栏展示词典通用释义并标注"(通用释义)"，状态切换功能不受影响。
- [ ] 删除文章 A 后，文章 B 中相同单词的状态和翻译保持不变。
- [ ] 色觉障碍模式下，红色/黄色单词除颜色外有额外视觉区分标记。
- [ ] 生词表中 `status` 字段正确区分 `new`、`reviewing`、`mastered` 三种状态；从未点击的词在生词表中无记录（`unseen` 不入库）。

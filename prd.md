# 产品需求文档

**项目名称：** Context Translation Layer  
**方向：** 浏览器插件优先的语境翻译与生词状态层  
**目标：** 用户在任意英文网页或最小阅读器中点击单词时，获得当前语境下的简短中文翻译，并把该词的学习状态同步到个人生词库。

---

## 1. 核心定位

这个产品不是阅读网站，也不承担内容运营和复杂阅读管理。

核心产品是一个可嵌入不同阅读场景的翻译层：

- 在外部网页中识别用户点击的英文单词。
- 提取当前句子或段落作为语境。
- 返回 2-6 个中文字的语境翻译。
- 记录用户对词的学习状态。
- 在不同文本中复用同一份个人生词状态。

当前 Web reader 只作为测试壳和 fallback reader：用户可以粘贴一段英文来验证核心体验，但它不是产品主体。

---

## 2. 用户故事

| # | 用户故事 |
|---|----------|
| US-01 | 作为英文网页阅读者，我希望点击网页上的单词时看到当前句子语境下的中文翻译，而不是词典所有释义。 |
| US-02 | 作为持续使用者，我希望同一位置的翻译结果能被缓存，避免重复等待和重复消耗 token。 |
| US-03 | 作为插件用户，我希望实时翻译和预翻译共享同一套语境翻译能力。 |
| US-04 | 作为测试用户，我希望打开侧边栏查看当前词、译文和来源句子。 |

---

## 3. MVP 范围

### In Scope

- 浏览器插件内容脚本原型：在网页正文中点击英文单词。
- 上下文抽取：优先取当前句子，必要时取邻近文本作为补充。
- `POST /api/v1/translate-word`：返回语境翻译。
- 位置级 annotation：同一词在不同语境中可以有不同翻译。
- 当前词详情侧边栏：显示语境译文与来源句子。
- 最小 Web reader：粘贴文章、打开文章、点击单词、删除文章。

### Out of Scope

- 与语境翻译层无关的阅读平台能力。
- 文章推荐。
- 复杂文章编辑。

---

## 4. 状态机

```
unseen -> translated
```

| 状态 | 含义 | 视觉 |
|------|------|------|
| `unseen` | 当前文本位置未翻译 | 普通文本 |
| `translated` | 当前文本位置已有语境译文 | 显示短译文 |

---

## 5. 核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/auth/register` | POST | 注册用户 |
| `/auth/login` | POST | 登录获取 token |
| `/api/v1/articles` | POST/GET | 最小 reader 的文本容器 |
| `/api/v1/articles/{id}` | GET/DELETE | 读取或删除文本容器 |
| `/api/v1/translate-word` | POST | 单词语境翻译并写入 annotation |

---

## 6. 验收标准

- 点击网页或 reader 中的 `bank`，在河流语境下返回“河岸”，在金融语境下返回“银行”。
- 同一词在不同位置的 annotation 互不覆盖。
- Web reader 不出现阅读平台功能入口。
